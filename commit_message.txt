
#61 Modification du système de confirmation par mail pour le reset de mdp

Qui: @PirLucas, basé sur le travail de @a-kalin0

Quoi: Jusqu'a maintenant, le mail de confirmation envoyé pour la réinitialisation de mot de passe fournissait un lien qui permettait d'authentifier l'utilisateur grace à un uuid et un token. 

Ce système de confirmation à été remplacé par un code de confirmation à entrer dans un formulaire. Ce code est envoyé par mail et doit etre copié-collé par l'utilisateur. 

Pourquoi: La confirmation par token utilisait les paramètres dans l'url ce que la saite parole de mr Schalkwijk nous à informé que c'était une très mauvaise chose.
La première altérnative (utiliser une requète POST) ne marche pas via un mail (ou alors c'est vraiment super dur)
Après réflexion, il se trouve qu'un code de vérification est aussi une manière de confirmer son identité valable et est beaucoup plus simple à implémenter tout en évitant le passage de paramètres dans l'url.


Comment: Le mail de vérification contient un code, la page qui confirme l'envoi du mail contiend un formulaire dans lequel on doit entrer le code. Si le code correspond à celui géneré par le serveur, l'utilisateur est redirigé vers la page de nouveau mot de passe.
